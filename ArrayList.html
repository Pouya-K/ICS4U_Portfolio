Arraylists are another form of lists or groups of locations of memory of the same data type, similar to arrays. The major difference between arraylists and arrays is that arraylists do not have a set size, and are easily resizeable. Additionally, arraylists can only hold objects (not primitive data types), which means that it will store int variables as Integers, and double variables as Double. Although the syntax may look a little different, Integers and Doubles still work the same way in ArrayLists, due to the concept of autoboxing. Autoboxing refers to the act of java automatically converting an Integer or a Double to an int or double, variable types that we are more familiar with. Arraylists require the java.util package, which needs to be imported at the start of the program. Similar to arrays, arraylists can be passed and returned to a method, and can be used in a for each loop using the same syntax.

With the flexibility of resizable arrays, arraylists come with many more methods. The .add() method will add a value to the end of the list, and will increase the size if necessary. Another variation of the .add() method will also take an index as a parameter. In this situation, the element will be added at that index, and elements from that index onward will be shifted down 1. On the other hand, the .set() method will take both an index AND an element, and will set that index to the specified element. Thirdly, the .remove() method will take an index as a parameter, and remove the element at the index, then proceed to move all the following elements back one. The last major method in the arraylist class is the .get() method, which is used to access the element at a specific index in the arraylist. The class comes with many more methods, which can be found here. Look at the code below for an example of the use of ArrayList and its methods. 

import java.util.*;
public class ArrayTesting {
    public static void main(String[] args) {
        ArrayList<String> testingArr = new ArrayList<String>(10);
        testingArr.add("I");
        testingArr.add("Love");
        testingArr.add("CS!");
        testingArr.remove(0);
        System.out.println(testingArr.get(0) + testingArr.get(1));
    }
}

Another unique feature that sets ArrayLists apart from arrays is their built in printing method. When a print is called on an ArrayList, the actualy values are printed as opposed to the address in memory pointing to the data, which is what happens when one tries to print an array. A common mistake that is seen when using arraylists comes when using the .remove() method. For example, the program below is intended to remove every instance of the word “Don’t” in an arraylist of Strings, but does not work as intended. This is because the size of the arraylist is decreased every time an element is reduced. As a result, elements may be skipped over when using a for loop. Instead, a while loop should be used, with the counter only being incremented if the element at that index is not removed. 

import java.util.*;
public class ArrayTesting {
    public static void main(String[] args) {
        ArrayList<String> testing = new ArrayList<String>();
        testing.add("I");
        testing.add("Don't");
        testing.add("Don't");
        testing.add("Love");
        testing.add("CS");
        for(int i = 0; i<testing.size(); i++){
            if(testing.get(i) == "Don't"){
                testing.remove(i);
            }
        }
        System.out.println(testing);
    }
}

Practice Questions

Consider the following code segment. 

int total = 3;
ArrayList<Integer> integerList = new ArrayList<Integer>();
for (int k = 7; k < 11; k++){
    integerList.add(k + 3);
}
for(Integer i : integerList){
    total += i;
    if(total % 2 == 1)
    {
        total -= 1;
    }
}
System.out.println(total);

What is printed as a result of executing the code segment.

(A)  34
(B)  37
(C)  46
(D)  47
(E)  49

Answer: C. After the initial loop, the content of ArrayList are: [10,11,12,13]. Let's consider what the for-each loop is doing. For each Integer in integerList, it will be added to total (which starts at 3), and if total is odd (total % 2 == 1), subtract one from it. Executing the for-each loop gives us 46 as our final answer.

Consider the following code segment. 

ArrayList<String> subjects = new ArrayList<String>();
subjects.add("French");
subjects.add("History");
subjects.set(1, "English");
subjects.add("Art");
subjects.remove(1); //Line 6
subjects.set(2, "Math");
subjects.add("Biology");
System.out.println(subjects);

What is printed as a result of executing the code segment?

(A)  [French, English, Math, Biology]
(B)  [French, Art, Biology]
(C)  [French, English, Art, Math, Biology]
(D)  [French, Math, Biology]
(E)  IndexOutOfBoundsException

Answer: E. By going through each call to the ArrayList methods, we reach the conclusion that subjects will have the contents [French, Art] after line 6 is compiled. When line 7 is called, trying to set index 2 to a value, an error will occur since index 2 does not yet exist in subjects. Therefore, trying it will generate an IndexOutOfBoundsException.

Assume that cities is an ArrayList<String> that has been correctly constructed and populated with the following items.

["Oakland", "Chicago", "Milwaukee", "Seattle", "Denver", "Boston"]

Consider the following code segment.

cities.remove(2);
cities.add("Detroit");
cities.remove(4);
cities.add("Cleveland");

What items does cities contain after executing the code segment?

(A)  ["Cleveland", "Detroit", "Oakland", "Chicago", "Seattle", "Denver", "Boston"]
(B)  ["Oakland", "Milwaukee", "Seattle", "Boston", "Detroit", "Cleveland"]
(C)  ["Oakland", "Chicago", "Seattle", "Boston", "Detroit", "Cleveland"]
(D)  ["Oakland", "Milwaukee", "Denver", "Boston", "Detroit", "Cleveland"]
(E)  ["Oakland", "Chicago", "Seattle", "Denver", "Detroit", "Cleveland"]

Answer: E. After removing at index 2, we will be left with ["Oakland", "Chicago", "Seattle", "Denver", "Boston"]. After adding "Detroit", we have ["Oakland", "Chicago", "Seattle", "Denver", "Boston", "Detroit"]. After the remove at index 4, we have ["Oakland", "Chicago", "Seattle", "Denver", "Detroit"]. Lastly, we add "Cleveland", making the final list ["Oakland", "Chicago", "Seattle", "Denver", "Detroit", "Cleveland"].
